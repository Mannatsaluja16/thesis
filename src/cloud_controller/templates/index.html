Architecture & About

AI-Based Hybrid Fault Tolerance Framework for Cloud Data Centers — thesis by Mannat Saluja.

System Architecture Diagram
Data and control flow across all five layers of the AI Fault Tolerance Framework.
Five-Layer Architecture
① User Layer
Web, mobile, and enterprise applications submit tasks via the REST API Gateway. No direct contact with infrastructure.
② Cloud Controller Layer
API Gateway (Flask) routes requests · Resource Manager maintains server registry · Task Scheduler delegates to the Energy-Aware Scheduler · Monitoring System polls every 30 s and feeds prediction windows.
③ AI Fault Prediction Layer
Data Collector pulls CloudWatch metrics or uses the Kaggle VM dataset · Preprocessor normalises and builds sliding-window sequences (length 10) · LSTM Model (3 layers, hidden=128) returns fault probability · Predict Engine runs real-time inference.
④ Fault Tolerance Layer
VM Migration — triggered at confidence > 0.5 · Adaptive Replication — only when confidence > 0.75 AND task = critical · Energy Scheduler — lowest energy-score server first · Recovery Manager — restores state and measures recovery time.
⑤ Physical Data Center Layer
10 simulated servers (8 CPUs, 16 GB each). In AWS: EC2 t3.micro instances monitored by CloudWatch. Storage and network abstracted by the Resource Manager.
Key Research Contributions
Proactive vs Reactive
LSTM predicts faults before they occur. Average recovery time reduced by 83% (24 s → 4 s).
Adaptive Replication
Conditional replication (confidence > 0.75 AND critical) reduces overhead ~40% vs always-on replication.
Energy-Aware Scheduling
Tasks placed on lowest energy-score servers first. Idle servers (<15% load) consolidated and suspended.

About This Project
Thesis: Fault Tolerance Mechanisms in Cloud Data Centers
Author: Mannat Saluja
Stack: AWS EC2, Python 3.10, PyTorch, Flask, Gunicorn, nginx
Repository: github.com/Mannatsaluja16/thesis

Combines LSTM fault prediction with adaptive fault tolerance to improve reliability and energy efficiency in cloud data centres. Proactive detection enables preemptive migration and selective replication only when risk is high.
